What kotoji gives you.

The one combination nothing else offers in a single place: self-hosting + MCP-native AI access + git-as-source-of-truth + a non-engineer-friendly editor.

← Back to the guide

Core features

Upload & serve

Drop a .zip of static files and get an instant URL — any asset-path style works.

Per-project subdomains

Each tool lives at your-tool.example.com, isolated on its own origin.

In-browser editing

A Monaco editor with diff view for quick fixes — no checkout, no local toolchain.

MCP-native

Connect Claude (or any MCP client) to list / read / write / publish your sites directly.

Per-user tokens

One token belongs to you and spans every project you're a member of; effective scope = token ∩ role.

git is the truth

Every save is a commit. History, diff and rollback come for free.

Draft → published

Work safely on branches; promote to production in one action. Each branch gets a preview URL.

Visibility tiers

Public, Members or Private per site — Private by default.

GitHub mirror & TLS

Optional off-box backup on every publish, and an opt-in Traefik overlay for automatic wildcard HTTPS.

Why self-host?

  • SaaS builders (Val Town, v0, Bolt, Lovable) are proprietary and can't be self-hosted.
  • Self-hosted PaaS (Coolify, Dokploy) assume a git-push CI flow — no in-browser editor, no AI integration.
  • kotoji combines all four pieces on your own server.
